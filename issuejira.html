Issue types in JIRA

Epic −

is a large user story which is to be broken into smaller storie.

Story −

a user story.

Task −

a task that needs to be done to achieve team’s goal.

Sub-Task −

is a piece of work which is to be done to complete the wholework.

Bug −

a problem that impairs or prevents the functions of the product.

Improvement −

an improvement or enhancement to an existing feature or task.

New Feature −

a new feature of the product, which is yet to be developed.



A typical JIRA issue of type 'bug' contains

Priority

Issue creator can set the priority to resolve the issue as High, Medium,Low, and Lowest.

Linked Issue

It links other issues that are either dependent on this issue or thisissue is dependent on them.

Labels

It is similar to Tag; it helps in filtering out specific types of issues..

Epic Link

An Issue creator can provide an epic link, if the issue belongs to any ofthose.

Sprint

The user can define in which sprint, this issue belongs to, when this issueshould be addressed.

Assignee

The person responsible to fix this issue. Assignee name can be entered by the issue creator.

Issue

User can link the issue by typing ID or summary of those that are related to the linked issue field.
